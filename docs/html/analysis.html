class symcirc.analysis.Analysis(circuit, method='tableau', symbolic=True, auto_eval=False, precision=6, sympy_ilt=True, use_symengine=False)¶

Bases: object

Main SymCirc class. When initialized it parses input netlist, and conducts the desired analysis which is then stored as a set of equations in the equation_matrix variable.

Parameters:

  • circuit (Circuit) – Loaded netlist file which contains the circuit description. If you intend to load from a file, use the utils.load_file() function.

  • analysis_type (str) – Analysis type identifier: “dc”, “ac”, “tf”, “tran”.

  • symbolic (bool) – False if you want your results evaluated with numerical values from the netlist.

  • method (str) –

  • auto_eval (bool) –

  • precision (int) –

  • sympy_ilt (bool) –

  • use_symengine (bool) –

ValueError – If the analysis_type argument is invalid.

class symcirc.analysis.Circuit(netlist, operating_points=None)¶

Bases: object

When initialized it parses the input netlist, and translates it into a list of components as defined in ‘symcirc.component’. This class implements methods for further circuit manipulation before analysis.

Parameters:

  • netlist (str) – string in netlist format which contains the circuit description. Note: If you intend to load from a file, use the utils.load_file() function.

  • operating_points (Dict[str, Dict[str, Any]]) –

    Use to pass numeric or symbolic operating point values for linearized components, if no values are specified the default model with symbolic values is used. Example: operating_points = {“Q1”: {“gm”: 74.5e-3, “gpi”: 232e-6, “gmu”: 1e-9, “go”: 22.5e-6, “gx”: 1.66}}

    The Q1 bjt will use these values in the model and expand the basic model accordingly.
